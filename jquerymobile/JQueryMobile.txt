************INFO ÚTIL************************************************
Worst thing you can do while creating hybrid mobile application is making it too large... Too much pages, too much non necessary content, it was clogging my mobile device memory, not to mention, user experience was … bad
> PÁGINAS ESENCIALES: EN INDEX.HTML - All essential pages needs to be part of first HTML file. That’s the file first loaded into the DOM. Every page loaded with it will permanently stay in memory so make sure you really really need them there. You can’t programmatically remove them from the DOM if AJAX is use as page handler.
> PÁGINAS NO ESENCIALES, CONTENIDO ESTÁTICO -  HTMLs separados - Separate non essential pages into other HTML pages (just remember the rule, one page per HTML file, only first HTML can hold more then one page). This should be rarely used pages, that usually contain static content, like ABOUT AS, CONTACT and similar.
> PÁGINAS DINÁMICAS - HTMLs separados con carga por JS (ej listviews) - Remove some pages completely, specially if we can generate them on our own. These are pages that will change over time, for example pages holding listview that’s refreshed with new content every few minutes.
